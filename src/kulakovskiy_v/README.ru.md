# BIOSEaRCH

**Поиск генов-кандидатов ишемического инсульта через анализ сети
белок-белковых взаимодействий и машинное обучение.**

English version: [README.md](README.md)

Репозиторий собирает воспроизводимую часть исследовательского проекта по
поиску генов-кандидатов, связанных с многофакторными заболеваниями (рабочий
пример — **ишемический инсульт**), путём анализа сети белок-белковых
взаимодействий (PPI) человека двумя дополняющими друг друга подходами:

1. **Прямой анализ сети** — ранжирование кандидатов по принципу
   «guilt by association», проверка значимости permutation-тестом
   (null-модель), сравнение с опубликованным алгоритмом поиска модулей
   болезни **DIAMOnD** и функциональная аннотация через **DAVID**.
2. **Машинное обучение на эмбеддингах узлов** — сеть кодируется в векторы
   узлов алгоритмом **node2vec**, после чего классификаторы (логистическая
   регрессия, SVM, дерево решений, LightGBM) обучаются на эмбеддинге и
   оценивают каждый ген по тому, насколько «инсультным» выглядит его
   положение в сети.

Работа выполнена в рамках научной программы **«Большие вызовы 2026»**
(направление *«Природоподобные и нейротехнологии»*, Образовательный центр
«Сириус») при научной поддержке **НИЦ «Курчатовский институт»**.

## Структура репозитория

```
BIOSEaRCH/
├── 01_network_pipeline/     Ветка 1 — прямой анализ сети + DIAMOnD + DAVID
│   ├── README.md            пошаговое руководство (англ.), README.ru.md — рус.
│   ├── FINDINGS.md          разбор последнего прогона (какие гены, DAVID)
│   ├── scripts/             пайплайн, пронумерован по порядку запуска (01…12)
│   └── external/            патченный чужой DIAMOnD (не наш код)
├── 02_ml_embeddings/        Ветка 2 — эмбеддинги node2vec + ML-классификаторы
│   ├── README.md            пошаговое руководство (англ.), README.ru.md — рус.
│   ├── scripts/             эмбеддинг, предсказание, сравнение, графики
│   └── data/emb8.csv        маленький пример эмбеддинга (остальные пересчитываются)
├── results/                 ключевые таблицы и картинки из обеих веток
├── data/README.md           где взять входные сети и списки генов
├── requirements.txt         зависимости Python
├── CITATION.cff             как ссылаться на работу
└── LICENSE                  MIT
```

У каждой ветки свой README с точными командами — начинать оттуда.

## Быстрый старт

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cd 01_network_pipeline   # или 02_ml_embeddings
less README.md
```

Входные сети и списки генов здесь **не** распространяются (принадлежат третьим
сторонам и/или большие) — как их получить, см. [data/README.md](data/README.md).
Маленький пример эмбеддинга (`02_ml_embeddings/data/emb8.csv`) включён, чтобы
ML-ветку можно было попробовать сразу.

## Воспроизводимость

- Все случайные операции засеяны (`seed=42`), прогоны повторяемы.
- Тяжёлые промежуточные выводы (null-модель на 10 000 реплик, полные
  эмбеддинги всех размерностей, большие матрицы) **не** коммитятся — они
  пересчитываются пайплайном. В `results/` лежат только компактные значимые
  результаты.
- Единственный чужой файл
  (`01_network_pipeline/external/DIAMOnD_with_pvalue.py`) — это DIAMOnD
  (Ghiassian, Menche & Barabási) с правкой в одну строку; он сохраняет
  оригинальную атрибуцию и условия (см. заголовок внутри файла).
