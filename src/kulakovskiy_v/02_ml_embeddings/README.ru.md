# ML на эмбеддингах PPI-сети

Пайплайн: **сеть → эмбеддинг узлов (node2vec) → предсказание генов-кандидатов
инсульта + визуализации**. Продолжение основного проекта (там были алгоритмы
на сети напрямую, тут — ML на векторных представлениях генов).

Всё запускается из `~/work/ML/embed` с активным venv:
```bash
source ~/.venvs/network/bin/activate
```

## Структура папки

```
embed/
├── 01_create_embedding.py       ШАГ 1: сеть -> эмбеддинг (node2vec)
├── 02_predict_and_visualize.py  ШАГ 2: предсказание кандидатов + 3 подхода ML
├── 03_plot_embedding.py         (утилита) t-SNE визуализация эмбеддинга (2D/3D)
├── data/
│   └── emb8.csv                 эмбеддинг всех генов (8-мерный)
├── results/                     все результаты (см. ниже)
├── net_01.txt                   маленькая тестовая сеть (для отладки)
└── _old/                        старые/черновые версии (не используются)
```

## ⚠️ Единый формат файла эмбеддинга

**CSV, БЕЗ заголовка, через запятую. 1-я колонка = gene_id, дальше признаки.**
```
4780,-0.29,-0.24,-0.28,0.25,0.58,-0.75,0.006,0.40
9817,-0.66,-0.12,0.05,0.23,0.19,-0.48,-0.31,0.05
...
```
Все три скрипта читают/пишут именно этот формат. (Раньше был рассинхрон:
node2vec писал через табы с заголовком, а ML-скрипты ждали запятые без
заголовка — из-за этого «ничего не работало». Сейчас всё сведено к одному
формату.)

## Шаг 1 — Создание эмбеддинга

```bash
python3 01_create_embedding.py -n /data/networks/HS-PI.clean.tsv -d 8 -o data/emb8.csv
```
- `-n` сеть (edgelist), `-d` размерность эмбеддинга, `-o` куда сохранить.
- `-d 8` — сейчас; на будущее посчитаем больше (`-d 16`, `-d 32`, `-d 128`) —
  чем больше, тем богаче признаки, дольше счёт. Даёт `data/emb16.csv` и т.д.
- Папка вывода создаётся сама. Долгий шаг (~10 мин на полной сети) —
  запускать лучше в `tmux`.

## Шаг 2б — Топ-100 кандидатов тремя методами (predict_top100.py)

Три алгоритма классификации, каждый выдаёт свой топ-100 генов-кандидатов.
Обучение на ПОЛНОМ эмбеддинге (все признаки). Один скрипт, меняется `--model`:

```bash
python3 predict_top100.py -e data/emb8.csv -g /data/genes/human-orthologs-02.txt --model logreg -o results/top100_logreg_8d.txt
python3 predict_top100.py -e data/emb8.csv -g /data/genes/human-orthologs-02.txt --model svm    -o results/top100_svm_8d.txt
python3 predict_top100.py -e data/emb8.csv -g /data/genes/human-orthologs-02.txt --model tree   -o results/top100_tree_8d.txt
```
- `--model logreg` = логистическая регрессия, `svm` = опорные векторы, `tree` = дерево решений.
- Для 16/32-мерного — то же самое, меняешь `-e data/emb16.csv` и имя `-o` (`..._16d.txt`).
- Итого получится матрица 3 алгоритма × N размерностей файлов топ-100.

### Пересечение топов (intersect_top100.py)

```bash
# сценарий А: один алгоритм, разные размерности (устойчивость к размерности)
python3 intersect_top100.py results/top100_svm_8d.txt results/top100_svm_16d.txt results/top100_svm_32d.txt -o results/intersect_svm.txt
# сценарий Б: одна размерность, все три алгоритма (гены, где сошлись все методы)
python3 intersect_top100.py results/top100_logreg_8d.txt results/top100_svm_8d.txt results/top100_tree_8d.txt -o results/intersect_8d.txt
```
Печатает строгое пересечение (гены во ВСЕХ списках) + раскладку «в скольки списках встретился ген».

> На 8-мерном пространстве три алгоритма почти не пересекаются (0 во всех трёх) —
> 8 измерений маловато, модели ловят разное. На 16/32 совпадений обычно больше.

## Шаг 2в — Предсказание кандидатов + 3 визуализации (02_predict_and_visualize.py)

```bash
python3 02_predict_and_visualize.py \
    -e data/emb8.csv \
    -g /data/genes/human-orthologs-02.txt \
    -o results
```
Что делает:
1. Размечает гены: **1** = seed-гены инсульта (`-g`), **0** = столько же
   генов, максимально далёких от seed в пространстве эмбеддинга (надёжный
   «не-инсульт»), **−1** = все остальные кандидаты.
2. Обучает 3 модели на классах {0,1} и строит по картинке на каждую:
   - **A. LightGBM** → тепловая карта вероятности `viz1_lightgbm_heatmap.png`
   - **B. SVM (rbf)** → нелинейные разделяющие зоны `viz2_svm_zones.png`
   - **C. LogReg** → линейная граница `viz3_logreg_zones.png`
3. Сохраняет таблицу всех генов по убыванию вероятности `genes_scored.csv`.

Выводит громкий пошаговый лог в консоль (ШАГ 1/6 … 6/6) + топ-10 кандидатов.

## Шаг 3 (утилита) — t-SNE картинка эмбеддинга

```bash
# интерактивный 3D (крутится в браузере)
python3 03_plot_embedding.py -e data/emb8.csv -o results/tsne_3d.html -g /data/genes/human-orthologs-02.txt -d 3
# статичный 2D/3D png
python3 03_plot_embedding.py -e data/emb8.csv -o results/tsne_2d.png -g /data/genes/human-orthologs-02.txt -d 2
```
`.html` → интерактивный plotly (крутить мышкой), `.png` → статичная картинка.
`-d 2` или `-d 3` — размерность проекции. seed-гены подсвечиваются красным.

## Что лежит в results/

| Файл | Что это |
|---|---|
| `genes_scored.csv` | все гены сети, отсортированы по вероятности «инсультный» (LightGBM) |
| `viz1_lightgbm_heatmap.png` | подход A — тепловая карта вероятности |
| `viz2_svm_zones.png` | подход B — зоны SVM |
| `viz3_logreg_zones.png` | подход C — граница логрегрессии |
| `tsne_2d.png` / `tsne_3d.html` | обзорная t-SNE картинка эмбеддинга |

## Как читать результат

- **seed-гены (чёрные ромбы)** должны кучковаться в «красной» зоне высокой
  вероятности, **дальний контроль (зелёные)** — в «синей». Если так — модель
  выучила осмысленное различие.
- **Кандидаты (серые)**, попавшие в красную зону и наверх `genes_scored.csv` —
  это гены, которые по своему положению в сети «похожи» на гены инсульта, но
  не были в исходном списке. Это и есть предсказание.
